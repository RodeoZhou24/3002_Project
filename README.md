# 动态定价模块  

## 项目简介  
本项目实现了一个用于电子产品的多因素动态定价策略模块。目标是结合成本、库存、市场需求、竞争对手定价等因素，动态调整商品售价，提高收益并优化库存周转。本模块采用 C++ 编写，并使用 CMake 进行构建。  

## 主要功能  
- **定价策略封装**：在 `PricingStrategy.h` 和 `PricingStrategy.cpp` 中实现核心定价算法，提供统一接口计算价格。  
- **可配置权重**：支持通过参数配置不同因素（如成本、库存、销量）的权重，以适应不同产品或市场策略。  
- **价格更新逻辑**：考虑线程安全和实时性，提供更新商品价格的机制，可与库存模块或销售模块对接。  
- **测试示例**：可根据需要编写简单的测试程序验证定价模型效果。  

## 文件结构  
- `PricingStrategy.h`：定价策略类的头文件，定义核心接口和内部数据结构。  
- `PricingStrategy.cpp`：定价策略类的实现文件，包含计算逻辑与参数调整。  
- `CMakeLists.txt`：项目的 CMake 配置，定义编译选项和依赖。  

## 构建指南  
项目采用 CMake 构建，需先安装 CMake 和支持 C++17 的编译器。构建步骤如下：  
1. 在项目根目录创建构建文件夹并进入：  
   ```bash  
   mkdir build && cd build  
   ```  
2. 运行 CMake 生成构建文件：  
   ```bash  
   cmake ..  
   ```  
3. 编译项目：  
   ```bash  
   cmake --build .  
   ```  
构建完成后，生成的可执行文件会位于 `build` 目录下（若有创建测试程序）。  

## 使用说明  
在使用该模块时，只需在其他代码中包含 `PricingStrategy.h`，实例化定价策略对象，并根据实际需求传入产品的成本、库存、销量等参数调用对应接口即可获取建议价格。可根据不同场景调整各因素权重，实现个性化定价。
